## 6. 启动脚本统一参数模板（待执行）

> 目标：将现场密钥集中到启动脚本传参，避免写入代码仓库配置文件。

- `k-dev-cli-api` 使用 `services.sh`（示例）：

```bash
JAVA_OPTS="$JAVA_OPTS \
 -Dsso.key=<SSO_KEY> \
 -Dencrypt.aes.secret=<AES_16_SECRET> \
 -Dencrypt.sm4.key=<SM4_KEY> \
 -Dencrypt.sm4.iv=<SM4_IV_16> \
 -Dapp.auth.tokenSecret=<APP_TOKEN_SECRET> \
 -Dapp.baidu.translate.securityKey=<BAIDU_TRANSLATE_KEY>"
```

- 说明：
  - `app.auth.tokenSecret` 变更会使历史登录 token 失效，建议安排在维护窗口执行。
  - 若未启用 SM4 数据源密文，可暂不传 `encrypt.sm4.key/iv`，但建议现场统一配置以避免后续故障。
  - 若现场未使用百度翻译，可不传 `app.baidu.translate.securityKey`。
